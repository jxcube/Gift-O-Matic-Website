Gift-O-Matic API

Jaxcube provides these APIs so developers can use data from Gift-o-Matic in new and extreme ways. At the moment we only have limited APIs, but this list will grow over time.

Gift-o-Matic Web APIs available

Item: Gift-o-Matic provides methods to fetch information for each Gift-o-Matic items.
Thread: Gift-o-Matic provides methods to fetch threads data in the database.
Post: Gift-o-Matic provides API calls to fetch posts data in the database.

Output Formats

All API calls take the form http://api.giftomaticapp.com/
Format:

JSON - The output will be returned in the JSON format

If you do not specify a format, your results will be returns in the JSON format.

Gift-o-Matic OpenID Provider

Gift-o-Matic can act as an OpenID provider. This allows your application to authenticate a user's Gift-o-MaticID without requiring them to enter their Gift-o-Matic username or password on your site (which would be a violation of the API Terms of Use.) Just download an OpenID library for your language and platform of choice and use http://Gift-o-Maticcommunity.com/openid as the provider. The returned Claimed ID will contain the user's 64-bit Gift-o-MaticID.

Jaxcube Brand and Links

Each page that uses the Gift-o-Matic Web API must contain a link to http://Gift-o-Maticpowered.com with the text "Powered by Gift-o-Matic". We suggest that you put this link in your footer so it is out of the way but still visible to interested users.
If you are using OpenID on your site, we request that you use one of the following buttons as your link to the Gift-o-Matic sign in page. These images represent the Gift-o-Matic brand to users, underscoring that Gift-o-Matic account credentials may be used to sign in to your site.

Policy

I. You will need to login to our website first in order to get our API. You will have to fill the needed fields with the real information and as many details as possible, so we have an idea of what you will do with our API.

II. If you need our users data, you will need to specify it first during the API request and give us the explanation details on why you would need to those. We are entitled to our users’ privacy, and in order to do that we would like to protect their data and we have the right to know what you will do with them.

III. If your application is integrated with Gift-O-Matic and a login into our system is needed, you will have to provide the option to login via OAuth or API key. You are not allowed to ask users for their email and password.

IV. You will have to protect and respect our users’ privacy. When integration is involved, you will have to state to the users what your application does to their account and what kind of information that you can retrieve from them.

V. We have the right to deny your request for our API if we feel that the information you provided is not enough, and we also have the right to block you from further using our API if we later find out that you are violating one of our policies.

VI. Since Gift-O-Matic is a free software, we will provide the API for free. But if you are intended on using it for commercial purpose, we will have the right to charge you for fees on the API.

VII. You will not put our logo or our name in such a way that makes it look like we endorse or affiliated with your application.

VIII. If at some point your third-party application that uses our API creates losses that affect Gift-O-Matic or our team, you will have to be responsible in all of the losses and we have the right to charge you on that.

IX. We own all of the rights in the services and API that we provide. You will not claim the API as your own. We also have the right to change the API with upcoming updates, and you are free to stop using if you feel that our API is no longer applicable to your application.

X. You will not make an application that copy how Gift-O-Matic works and looks using our API.

XI. You will not use our API to make an application that violates any law, rights, or regulations. You will also not make an application that is intended to spam or harass our user.
